# Contributing to DAV Practicals

Thank you for your interest in contributing to this Data Visualization & Analytics practicals repository!

## How to Contribute

### Adding New Practicals

If you'd like to add a new practical:

1. **Fork the repository**
2. **Create a new branch** for your practical

   - Learning outcomes
   - Step-by-step instructions
   - Expected output

5. **Test your code** to ensure it runs without errors

6. **Submit a Pull Request** with a clear description

### Improving Existing Practicals

To improve existing practicals:

1. Fork and create a new branch
2. Make your improvements
3. Test thoroughly
4. Submit a PR explaining the improvements

### Adding Resources

To add helpful resources:

1. Add files to the `resources/` directory
2. Update the resources README.md
3. Ensure the resource is properly documented

### Reporting Issues

If you find a bug or have a suggestion:

1. Check if the issue already exists
2. Create a new issue with:
   - Clear description
   - Steps to reproduce (for bugs)
   - Expected vs actual behavior
   - Your environment details

## Code Style Guidelines

- **Python**: Follow PEP 8 style guidelines
- **Comments**: Add comments for complex logic
- **Docstrings**: Use docstrings for functions
- **Naming**: Use descriptive variable names
- **Formatting**: Use consistent indentation (4 spaces)

## Commit Message Guidelines

- Use clear, descriptive commit messages
- Start with a verb (Add, Update, Fix, Remove, etc.)
- Keep the first line under 50 characters
- Add details in the body if needed

Example:
```
Add practical 05: Time series analysis

- Include sample stock market data
- Implement moving averages
- Add trend visualization
```

## Testing

Before submitting:
- Run your code to ensure it works
- Check for syntax errors
- Verify all required files are included
- Test with the provided datasets
